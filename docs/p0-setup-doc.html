DÉCISIONS ARCHITECTURALES — POURQUOI
SUJET
DÉCISION
RAISON
Package Manager
pnpm 10
workspace:* protocol, store global, isolation stricte, security by default sur les postinstall scripts
OS dev
WSL2 Ubuntu 24.04
Docker tourne déjà en Linux en backend WSL2. Husky, scripts shell, Metro bundler fonctionnent sans adaptation. Parité avec CI GitHub Actions (Ubuntu).
Fichiers projet
/home/kylian/ (pas /mnt/d/)
I/O Docker sur /mnt/ = filesystem Windows, lent. /home/ = filesystem WSL2 natif, performances comparables à Linux réel.
Node version
Node 22 LTS via fnm
Node 22 est LTS depuis oct 2024. fnm supporte .nvmrc nativement + 40x plus rapide que nvm. Auto-switch avec --use-on-cd.
ESLint config
Flat config (eslint.config.mjs)
ESLint v10 (fév. 2026) supprime définitivement .eslintrc. Pas de choix. Le package typescript-eslint unifie parser + plugin.
BDD locale vs cloud
Docker local en dev, Supabase uniquement en staging/prod
Travail offline possible. Migrations rapides et destructives (prisma migrate reset) sans risquer une BDD partagée. Latence réseau éliminée.
Logging applicatif
Logger @nestjs/common (jamais console.log)
Quand Pino est branché via app.useLogger(), tout le code utilisant Logger NestJS bénéficie automatiquement de Pino sans modification. console.log = logs non structurés, impossibles à parser ou envoyer vers Axiom.
Commits
Conventional Commits (commitlint)
Format type(scope): desc. Permet génération automatique de changelog. Historique Git = documentation lisible. Bloquant via hook commit-msg.
