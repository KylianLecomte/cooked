DÉCISION

workspace:* protocol, store global, isolation stricte, security by default sur les postinstall scripts
OS dev

I/O Docker sur /mnt/ = filesystem Windows, lent. /home/ = filesystem WSL2 natif, performances comparables à Linux réel.
Node version

Node 24 LTS actif. fnm supporte .nvmrc nativement + 40x plus rapide que nvm. Auto-switch avec --use-on-cd. .nvmrc contient "24".
ESLint config

ESLint v10 supprime définitivement .eslintrc. Config CLI NestJS conservée (recommendedTypeChecked + projectService) — plus complète que la config de base.
BDD locale vs cloud
Docker local en dev, Supabase uniquement en staging/prod
Travail offline possible. Migrations rapides et destructives sans risquer une BDD partagée. Latence réseau éliminée en dev.
Test runner
Vitest 3.2.4 + unplugin-swc
esbuild (défaut Vitest) ne supporte pas emitDecoratorMetadata → injection NestJS silencieusement cassée. SWC supporte les décorateurs. module: es6 dans swc.vite().
Commits
Conventional Commits (commitlint)
Format type(scope): desc. Historique Git lisible, génération de changelog automatique possible. Bloquant via hook commit-msg.
Stratégie de branches
GitLab Flow — environment branches
Flux unidirectionnel : feature/* → main → staging → production. Chaque branche permanente représente un environnement réel. Toutes protégées — merge uniquement via PR + CI verte, jamais de commit direct.

Gitflow écarté : back-merges release→develop trop verbeux. GitHub Flow écarté : pas conçu pour multi-environnements. Démarrage progressif : seule main existe pour l'instant, staging et production ajoutées en P8 quand les environnements existent.

Versioning : tags SemVer sur production uniquement — déclenche EAS Build (P8-03). Hotfix : branche depuis production, PR sur production, report obligatoire sur main via PR séparée.
